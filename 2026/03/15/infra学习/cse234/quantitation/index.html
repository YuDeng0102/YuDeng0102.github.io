gpu memory 占用分析

gpu的memory是有限的，在训练大模型时，需要考虑如何在有限的memory中存储模型参数和中间结果,确保峰值memory占用不超过显存容量。gpu的memory开销主要有三个：

  • 模型参数：模型的权重和偏置等参数
  • 激活值：前向传播中间结果
  • 优化器状态：优化器在训练过程中需要存储一些状态，如梯度、动量、学习率等

模型参数

以175b的gpt3为例，模型参数使用f16存储

模型参数占用显存为175B*2=350G

激活值

以transformer为例，简化后的激活值大小为 batch_size * seq_len * hidden_size * sizeof(element)

一层的transformer的激活值就是78G

优化器状态

以adam优化器为例，每个参数需要存储自身的梯度，移动平均数以及移动方差，因此假设原来的参数量为N,优化器存储的显存占用为N*3*sizeof(element)

减小内存策略

主要分成单机和并行两类，这篇文章主要关注单机

减少激活值内存占用

  • Activation Checkpointing

    activation value只有当需要反向传播时才会被计算，因此可以将部分激活值前向传播后直接释放，反向传播时再计算。如图所示

    在这个过程中，需要平衡计算成本和显存占用,这种方法的具体名称叫Activation Checkpointing，pytoch就提供了对应的api

  • Gradient Accumulation

    如果显存不足以存储一个batch的激活值，可以设置micro batch，每次只计算一个micro batch的激活值，反向传播时再累加梯度。比如batch size为128，micro batch size为4，那么每次只计算128/4=32个样本的激活值，反向传播时再累加32个样本的梯度。这样用时间换空间。

  • cpu swap
    很简单，就是将一部分数据从显存swap到cpu内存，等需要时再swap回显存。

量化

量化也是一种减少内存占用的方式，把模型的参数、激活值、梯度等从高精度的浮点数转换成低精度的整数或定点数，从而减少显存占用。这节课介绍了两种经典的量化方法：K-means 量化和线性量化。

k-means 量化

将个不同的浮点数分成个聚类中心，每个聚类中心仍然用原来的位数表示，原张量中的每个数据变成了一个位的整数，表示它所属的聚类中心的索引，同时存储一个聚类中心的表，用于将整数索引映射回浮点数。
